# 拷问.skill — 方案与内容系统性压力测试框架

> **拷问不是无痛的润色，而是高强度的压力测试。**

本框架提炼自高标准品控（Quality Assurance）的底层逻辑，专为业务方案、产品策划、商业决策、方法论文章提供系统化的八维度拷问体系。

## 八维度框架

| 等级 | 角度 | 核心目的 |
|:---|:---|:---|
| 🟡 等级1：基本盘 | 前提、边界、语境 | 排除致命逻辑硬伤，防止"沙滩上建高楼" |
| 🟠 等级2：深度挖掘 | 区分、反例、深挖 | 拉开认知差距，拒绝平庸方案 |
| 🔴 等级3：落地安全 | 实操、代价 | 确保可执行，防止次生灾害 |

## 输出物

每次拷问生成结构化的《方案/内容拷问报告》，包含：
- 一致性评级（自洽性校验）
- 时间轴推演（3/6/12个月）
- 三个等级的逐项诊断
- 综合品控评级（🟢保留 / 🟡修复 / 🔴重写）
- 核心优化路径

## 适用场景

- 产品与业务方案品控（评审前自我排查）
- 专业内容发布前质检（排查偏见与漏洞）
- 决策可行性压力测试（暗坑与次生灾害）
- 自洽性校验（防止"今天的方案打脸昨天的规划"）
