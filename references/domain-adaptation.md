# 拷问框架领域适配指南

> 实战积累：当拷问框架应用于其原始设计域之外的场景时，如何补充领域知识、如何调整输出结构。

---

## 背景

拷问.skill 的八维度框架、场景示例和输出模板主要面向三类对象：

- **业务/产品方案**（定价策略、功能规划、运营方案）
- **商业决策**（投资分析、市场进入、组织变革）
- **方法论/内容文章**（观点输出、知识分享）

当应用于其他领域（如**工程施工策划、政府项目可研、专业技术方案**）时，框架本身依然有效，但需要在执行时做以下补充。

---

## 一、为通用框架补充领域知识

不要输出"框架不够用"的评论。用户认为框架是通用的——你的职责是**在框架之上补充必要的领域视角**，而不是指出框架的盲区。

### 工程施工类方案

以下是施工组织/项目策划方案在拷问中容易暴露的领域特有盲区：

| 拷问维度 | 领域特有问题示例 | 需要补充的视角 |
|---------|----------------|---------------|
| 前提拷问 | "公司能承受启动阶段的垫资压力吗？" | 付款节点节奏 → 资金峰值 → 公司财务承受力 |
| 边界拷问 | "铝模+爬架方案假设标准层无变化" | 设计变更对工艺选型的推翻风险 |
| 深挖拷问 | "为什么利润率只有5%?" | 材料调差条款敞口、结算送审风险、保修金占用 |
| 反例拷问 | "类似安置房项目因钢材涨价亏损" | 材料价格波动、劳务短缺、政府流程延迟 |
| 实操拷问 | "加强变更签证管理"是空话 | 需要补充：谁负责识别变更、谁编制签证、谁来跟踪审批 |
| 代价拷问 | "付款延迟导致资金链断裂" | 政府项目拨款延迟是常态，需要做情景模拟 |

### 通用原则

对于任何非标准领域的方案/内容，在执行拷问前先问自己：
- 这个领域有哪些**隐性成本**（质量成本、合规成本、机会成本）？
- 有哪些**行业特有的风险条款**（调差条款、罚款条款、支付条件）？
- 有哪些**默认会被从业者接受但实际上很脆弱的假设**（"甲方会按时付款"、"劳动力充足"、"材料价格稳定"）？

## 二、输出结构规则

### 规则1：一次拷问 = 一份文档，聚焦于被拷问的对象

**错误做法**（本session的初始错误）：
```
文档分三部分：
- 第一部分：拷问skill的意见与建议
- 第二部分：拷问详细内容
- 第三部分：我的洞察
```
用户不需要关于"工具"的分析，只需要关于"被拷问的对象"的分析。

**正确做法**：
```
文档结构：
- 综合判断（一句话定性）
- 🔴 必须修正（致命问题）
- 🟡 建议优化（提升项）
- 🟢 已做对（确认项）
- 总结与行动建议
```

### 规则2：领域适配是内功，不是输出的主题

- 如果发现框架对当前领域覆盖不足 → 用LLM领域知识**补上缺口**，而不是在输出中抱怨框架不够用
- 如果框架的某个维度在当前领域特别出彩 → 正常使用，不需要特别标注"这个维度很有效"
- 合成输出时，把所有来源（初印象、框架分析、领域知识）合并到一个连贯的建议中

### 规则3：用户要的是"意见与建议"，不是"分析报告"

- 当用户说"给意见/建议"时，优先输出**行动项**（"补一名预算员"），而不是分析过程（"通过5-Whys穿透发现..."）
- 分析过程可以缩减到一句话说明理由，不要展开论证链条
- 三色标记（🔴🟡🟢）优先于文字段落

## 三、典型场景备忘

### 施工项目策划方案拷问

**高价值问题清单**（可直接复用的问题模板）：

```
前提类：
- 项目启动资金从哪里来？公司能支撑到什么程度的垫资？
- 材料调差条款的敞口有多大？近几年市场行情下这个敞口会触发吗？

边界类：
- 铝模/爬架方案是否依赖"标准层无变化"这个假设？如果设计变更呢？
- 施工通道方案是否依赖于"中间地库不开挖"？如果地质条件要求必须提前开挖呢？

反例类：
- 南通/如东地区同类安置房项目，有没有亏损案例？亏在哪里？
- 有没有项目因为结算超审价5%被处罚的案例？

实操类：
- 结算编制人员在哪儿？180天能交付吗？
- 变更签证的管理流程是什么？谁负责？
- 现金流压力测试跑过没有？最坏情景是什么？
```

### 政府项目/安置房方案特有盲区

- 政府审批流程长 → 签证确认周期长 → 资金回收滞后
- 设计变更走政府部门会签 → 周期不可控
- 工程质量目标"合格"即可 → 但"合格"在安置房场景下可能被舆论放大
- 付款节点与政府财政拨付时间脱节是常态

### 政企博弈函件/法律文书（2026-05-28 实战沉淀）

**场景特征**：企业向政府/管委会/s局等机关发出的回函、沟通函、情况说明等。这不是纯法律论证，而是在行政权力不对称下的**博弈修辞**。

**拷问框架的四个关键注入视角**：

```
① 识别真正的读者
   └─ 收件人不一定是最终决策者。函件可能是"转上去"的中转材料。
      问自己：这封函被送到更高层级（巡视组、审计、信访）后，会怎么被解读？

② 区分"法律正确"与"博弈有效"
   └─ 纯法律论证在不对称博弈中可能被对方选择性执行。
      对方可以用"合同相对性""法人独立"等形式合法理由驳回诉求，
      但同时自己却在跨主体扣款、跨主体监管——这是选择性执法，不是法律公平。

③ 识别对方的"双标"漏洞
   └─ 对方自己是否在同一标准下行事？
      如果对方在主张A时强调形式合法、在行为B时却跨主体操作，
      这个"不一致"本身就是最有力的反击论点。
      例：管委会跨主体扣兄弟公司2000万→却主张催租需严守法人独立。

④ 函件的四层功能模型
   └─ 承认义务为锚点      → 不给对方"拒付"的话柄
      揭示双标为矛头      → 指出对方的不一致行为
      承诺分期为安全阀    → 对方有台阶下、我方有操作空间
      证据留存为底线      → 为将来升级渠道（纪检/审计/信访）做准备
```

**高价值拷问清单**：

```
前提类：
- 这封函的真正读者是谁？除了盖章的收件人，会不会被转给纪委、审计、上级政府？
- 我方论证依赖的前提，对方自己是否遵守同一标准？

边界类：
- 在什么情况下这封函会被定性为"推诿"或"恶意拖延"？
- 这封函发出后，对方可能抓住哪个词/哪句话做文章？

反例类：
- 同类政企纠纷中，单纯讲法理的企业赢过吗？靠什么赢的？
- 有没有因为函件措辞过激而被对方抓住把柄反击的案例？

代价类：
- 这封函的强硬程度会不会关闭谈判窗口？在哪些场景下需要"硬中有软"？
- "向上级反映"的施压点，精度够吗？纪检、审计、行政诉讼三条线，哪条最有效？
```

**联合审查的迭代验证方法论**（2026-05-28 管委会函件实战）

当拷问后使用联合审查（多部门AI审查系统）做二次验证时，审查结果需要一套解读框架：

```
步骤1：区分"否决什么"和"没否决什么"
  └─ 联合审查的三个部门都说"从保证金划付租金"不可行 → 操作方式被否
  └─ 但没有一个部门说"2000万保证金不该退还" → 实体诉求没被否
  └─ 结论：保留实体诉求，换操作路径（两步走：先退还→再协议处理）

步骤2：从否决中寻找"反向指导"
  └─ 纪委监委利益冲突审查说："应签《债务抵销及支付协议》明确三方关系"
  └─ 这不是批评，这是指明了合法合规的路径
  └─ 结论：联合审查说"不能这样走"的时候，往往暗示了"应该那样走"

步骤3：对比多轮审查结果，验证迭代方向
  └─ 第一轮：财政局❌、纪委🟡、司法局❌
  └─ 修改后第二轮：财政局不涉及✅、纪委不涉及✅、司法局🟡
  └─ 结论：三个部门都改善了——说明修改方向对了
```

**关键认知**：联合审查站在"形式合规"视角，而用户站在"真实博弈"视角。当两者冲突时：
- 形式合规的批评必须尊重（否则函件被直接退回）
- 但不可以被形式合规牵着放弃核心诉求
- 正确的做法是：**保留实体目标，重新设计操作路径**（如从"直接划付"改为"先退还再协议"）

**经典坑点（来自实战）**：

- 不要在主函中写"贵单位一直采取穿透式一体化监管"而不附具体事例——对方会反问"请举证"。至少加括号列举具体表现。
- "舆情催收""过激手段"等定性措辞不要出现在正式函件中——既让对方可以反驳（"我方从未使用"），又把法律诉讼也定性为"过激"，自损立场。
- "债务抵销"在联合署名函中看似有力，但抵销要求当事人同一——如果管委会和居委会不是同一主体，抵销主张在法理上不成立。改为"恳请协调划付/安排结算"。
- 联合署名（多家关联公司共同发函）是封死"合同相对性"推诿路径的有效手段，但需确认各公司印章可用。

**输出结构**：

政企函件的拷问输出应优先采用"三色标记表"（🔴必须修正 / 🟡建议优化 / 🟢已做对），而非长文分析。每条意见附带一句理由即可，不展开论证链条。
